# AutoAI 拡張機能セットアップガイド

## 別のパソコンで使用する際の設定手順

### 1. 拡張機能のインストール

1. Chrome ブラウザで `chrome://extensions/` を開く
2. 右上の「デベロッパーモード」を有効にする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. AutoAIフォルダを選択

### 2. Google認証の設定

#### 初回認証
1. 拡張機能のアイコンをクリック
2. 「認証」ボタンをクリック
3. Googleアカウントでログイン
4. 必要な権限を許可
   - Google スプレッドシートへのアクセス
   - Google ドキュメントへのアクセス
   - Google ドライブへのアクセス

#### トラブルシューティング

**「スプレッドシート読み込みエラー」が発生する場合：**

1. **認証の再実行**
   - Chrome の設定 → プライバシーとセキュリティ → サイトの設定
   - 「すべてのサイトの権限と設定を表示」
   - `chrome-extension://` で始まる AutoAI の URL を探す
   - 保存されているデータをクリア
   - 拡張機能から再度認証

2. **スプレッドシートの共有設定確認**
   - 対象のGoogleスプレッドシートを開く
   - 右上の「共有」ボタンをクリック
   - リンクを知っている全員が「閲覧可」または「編集可」に設定
   - または、使用するGoogleアカウントに直接共有

3. **拡張機能の権限確認**
   - `chrome://extensions/` で AutoAI の詳細を開く
   - 「サイトへのアクセス」が「すべてのサイト」になっているか確認

### 3. 必要な権限

拡張機能が正常に動作するには、以下の権限が必要です：

- **Google APIs**
  - sheets.googleapis.com
  - docs.google.com
  - www.googleapis.com

- **AI サービス**
  - chatgpt.com
  - claude.ai
  - gemini.google.com
  - genspark.com

### 4. よくある問題と解決方法

#### Q: 認証ボタンを押しても反応しない
A: ポップアップブロッカーが有効になっている可能性があります。Chrome の設定でポップアップを許可してください。

#### Q: スプレッドシートが読み込めない
A: 以下を確認してください：
1. Googleアカウントでログインしているか
2. スプレッドシートへのアクセス権限があるか
3. URLが正しいフォーマットか（例：https://docs.google.com/spreadsheets/d/[ID]/edit）

#### Q: 「認証エラー」が表示される
A: 以下の手順で解決できます：
1. Chrome のキャッシュとCookieをクリア
2. 拡張機能を一度無効化して再度有効化
3. 拡張機能を再インストール

### 5. 開発者向け情報

#### OAuth2.0 クライアントIDについて

現在の `manifest.json` に設定されているクライアントID:
```
262291163420-fj2jfie1cmb63md9nnu4ofdkr9ku1u3o.apps.googleusercontent.com
```

**独自のクライアントIDを使用する場合：**

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 新しいプロジェクトを作成または既存のプロジェクトを選択
3. APIとサービス → 認証情報 → OAuth 2.0 クライアントIDを作成
4. アプリケーションの種類: Chrome アプリ
5. アプリケーションID: 拡張機能のID（chrome://extensions/ で確認）
6. 生成されたクライアントIDを `manifest.json` の `oauth2.client_id` に設定

#### 必要なGoogle API

以下のAPIを有効にする必要があります：
- Google Sheets API
- Google Docs API
- Google Drive API

### 6. サポート

問題が解決しない場合は、以下の情報と共に報告してください：
- エラーメッセージの詳細
- Chrome のバージョン
